## 5. 탭 3: 재고 변동 이력

### 5.0 재고 변동 이력 생성 규칙

**기본 원칙:**
- 재고 수량은 Location(위치) 기준으로 계산됨
- Location이 실제 창고(본사창고, 강남창고 등)인 자재만 재고로 집계
- Location이 현장, 불량창고, null인 자재는 재고에서 제외

**이력 생성 시점:**

1. **입고 등록 시**
   - 입고 등록 완료 즉시 재고 변동 이력 생성
   - 입고 수량만큼 재고 즉시 증가
   - Location: 입고 창고로 설정
   - 사유/상태: "-" (검수 전)

2. **검수 완료 시**
   - 기존 입고 이력 업데이트 (새 이력 생성 안 함)
   - 사유/상태: "-" → "검수: 정상 N, 불량 N"
   - 불량품 처리: Location을 "불량창고"로 변경
   - 재고 수량: 정상품만 집계 (불량창고는 재고 제외)

3. **출고 등록 시**
   - 출고 등록 완료 즉시 재고 변동 이력 생성
   - 출고 수량만큼 재고 즉시 감소
   - Location: 출고 창고 → 현장으로 변경

4. **재고 조정 시**
   - 재고 조정 완료 즉시 재고 변동 이력 생성
   - 조정 수량만큼 재고 증감
   - Location: 조정 대상 창고

5. **시리얼 상태 변경 시**
   - Location 변경이 있는 경우에만 재고 변동 이력 생성
   - 예: 정상 → 기능불량 (창고 → 불량창고)
   - Location 변경이 없는 경우: 시리얼 이력에만 기록
   - 예: 정상 → 외관파손 (창고 유지)

**불량품 처리 방식:**
- 검수에서 불량 판정된 자재는 Location을 "불량창고"로 변경
- 불량창고는 재고 집계에서 제외
- 반품/폐기 처리 시 별도 이력 생성 (향후 구현)

**이력 업데이트 vs 추가 생성:**
- 입고 검수: 기존 이력 업데이트 (사유/상태만 수정)
- 재고 조정: 새 이력 생성
- 출고/회수: 새 이력 생성
- 시리얼 상태 변경: 새 이력 생성 (Location 변경 시)

### 5.1 검색/필터 영역

**필터 옵션:**

- 기간 (시작일 ~ 종료일)
- 유형 전체 / 입고 / 출고 / 회수 / 재출고 / 파손 / 분실
- 소유구분 전체 / 소유 / 임대 / 현장전용
- 품질 전체 / 신품 / 중고
- 자재명/시리얼 검색
- 현장 필터

### 5.2 요약 영역

```
입고: 120건 | 출고: 85건 | 회수: 23건 | 재출고: 8건 | 파손/분실: 5건
```

**참조번호 클릭:**

- 현재 화면 유지한 채 해당 문서의 상세 모달 오픈
- 참조번호 유형별 동작:
  - PO-YYYY-NNN (구매 번호) → 구매 상세 모달
  - IT-YYMMDD-NN (발주 번호) → 발주 상세 모달
  - IN-YYYY-NNN (입고 번호) → 입고 상세 모달
  - OUT-YYYY-NNN (출고 번호) → 출고 상세 모달
  - AS-YYYY-NNN (AS 번호) → AS 상세 모달
- 모달 닫으면 재고 변동 이력 화면으로 복귀

**사유/상태 컬럼 표시 규칙:**

모든 이동유형에 대해 **"[카테고리]: [상세내용]"** 형식으로 통일하여 표시합니다.

1. **입고 (구매)**
   - 검수 정상: "-" (사유 없음)
   - 검수 일부불량: "검수: 정상 N, 불량 N"
   - 검수 전량불량: "검수: 전량불량 N"

2. **입고 (현장회수)**
   - 형식: "회수: [사유]"
   - 예: "회수: 미사용", "회수: 고장", "회수: 파손", "회수: 과다출고"

3. **출고**
   - 표시: "-" (사유 없음)

4. **AS입고**
   - 형식: "AS: [문제유형]"
   - 예: "AS: 초기불량", "AS: 고장", "AS: 파손", "AS: 오작동"

5. **재출고 (AS)**
   - 형식: "재출고: [처리결과]"
   - 예: "재출고: 수리완료", "재출고: 교체"

6. **재고조정**
   - 형식: "조정: [사유]"
   - 예: "조정: 실사", "조정: 분실", "조정: 파손", "조정: 오입력"

7. **분실**
   - 형식: "분실: [사유]"
   - 예: "분실: 현장분실", "분실: 운송중분실"

8. **파손**
   - 형식: "파손: [사유]"
   - 예: "파손: 외관파손", "파손: 낙하파손"

**사유/상태 데이터 입력 출처:**

| 이동유형 | 입력 화면 | 입력 필드 | 비고 |
| --- | --- | --- | --- |
| 입고 (구매) | 입고 관리 > 입고 검수 | 검수 결과 (필수) | 정상/일부불량/전량불량 |
| 입고 (현장회수) | 입고 관리 > 입고 등록 | 회수 사유 (필수) | 미사용, 고장, 파손 등 |
| 출고 | 출고 관리 > 출고 등록 | - | 사유 불필요 |
| AS입고 | AS 관리 > AS 접수 | 문제 유형 (필수) | 초기불량, 고장, 파손 등 |
| AS재출고 | 출고 관리 > 출고 등록 > AS 출고자재 선택 | 교체출고 여부 | 교체출고, 재출고 등 |
| 재고조정 | 재고 관리 > 재고 조정 | 조정 사유 (필수) | 실사, 분실, 파손, 오입력 등 |

## 8. 시리얼 상태 변경 모달

### 8.1 오픈 조건

- 시리얼 목록의 [상태변경] 버튼 클릭
- 시리얼 상세 모달의 [상태 변경] 버튼 클릭

### 8.2 표시 정보 (읽기 전용)

- 시리얼 번호
- 자재명
- 현재 상태
- 현재 위치 (Location)

### 8.3 입력 필드

**변경할 상태* (필수, 라디오 버튼):**

- 정상 / 외관 파손 / 기능 불량 / 완전 파손 / 분실 / 폐기

**변경 사유* (필수):**

- textarea, 3줄
- placeholder: "상태 변경 사유를 입력하세요"

**변경 일시:**

- datetime-local, 기본값: 현재 일시

### 8.4 상태 변경 시 Location 자동 결정

시리얼 상태 변경 시 Location이 자동으로 결정되며, 이는 일반 자재의 입고/출고/검수 프로세스와 동일한 원칙을 따릅니다.

**Location 자동 변경 규칙:**

| 변경 전 상태 | 변경 후 상태 | Location 변경 | 재고 변동 | 이력 생성 |
|------------|------------|--------------|----------|----------|
| 정상 | 외관 파손 | 유지 (창고) | 없음 | X |
| 정상 | 기능 불량 | 창고 → 불량창고 | -1 | O |
| 정상 | 완전 파손 | 창고 → 불량창고 | -1 | O |
| 정상 | 분실 | 창고 → null | -1 | O |
| 정상 | 폐기 | 창고 → null | -1 | O |
| 기능불량 | 정상 | 불량창고 → 본사창고 | +1 | O |
| 완전파손 | 정상 | 불량창고 → 본사창고 | +1 | O |
| 외관파손 | 기능불량 | 창고 → 불량창고 | -1 | O |
| 외관파손 | 정상 | 유지 (창고) | 없음 | X |

**Location 결정 원칙:**

1. **정상/외관파손**: 실제 창고 (본사창고, 강남창고 등) - 재고 포함
2. **기능불량/완전파손**: 불량창고 - 재고 제외
3. **분실/폐기**: null (위치 없음) - 재고 제외

### 8.5 [상태 변경] 버튼

**유효성 검증:**

1. 변경할 상태 선택 확인
   - 미선택 시: "변경할 상태를 선택해주세요" 알림
2. 변경 사유 입력 확인
   - 미입력 시: "변경 사유를 입력해주세요" 알림
3. 현재 상태와 동일한 상태 선택 확인
   - 동일 시: "현재 상태와 동일한 상태로 변경할 수 없습니다" 알림

**동작 순서:**

1. 필수 필드 검증
2. 시리얼 상태 변경 처리
3. Location 자동 변경 (8.4 규칙 적용):
   - 정상 → 기능불량/완전파손: Location → 불량창고
   - 정상 → 분실/폐기: Location → null
   - 기능불량/완전파손 → 정상: Location → 본사창고
   - 정상 ↔ 외관파손: Location 유지
4. 재고 변동 이력 생성 (Location 변경 시에만):
   - Location 변경 있음:
     - 이동유형: 재고조정 (기능불량/완전파손), 분실 (분실/폐기)
     - 수량: +1 (재고 증가) 또는 -1 (재고 감소)
     - 이동구간: 이전 Location → 새 Location
     - 사유/상태: "상태변경: [변경 후 상태]" 또는 "분실: [사유]"
     - 시리얼 번호 기록
   - Location 변경 없음:
     - 재고 변동 이력 생성 안 함
5. 시리얼 이동 이력 기록 (모든 상태 변경):
   - 변경 전 상태 → 변경 후 상태
   - 변경 사유
   - 변경 일시
   - 담당자
6. 성공 알림: "시리얼 상태가 변경되었습니다"
7. 모달 닫기, 목록 갱신

**재고 변동 이력 예시:**

```
| 일시 | 이동유형 | 자재명 | 수량 | 이동구간 | 참조번호 | 사유/상태 |
| 2024-02-15 10:00 | 재고조정 | IP카메라 2MP | -1개 | 본사창고 → 불량창고 | - | 상태변경: 기능불량 |
| 2024-02-15 10:00 | 분실 | IP카메라 2MP | -1개 | 본사창고 → - | - | 분실: 현장 분실 확인 |
| 2024-02-20 14:00 | 재고조정 | IP카메라 2MP | +1개 | 불량창고 → 본사창고 | - | 상태변경: 정상 |
```

### 8.6 [취소] 버튼

- 모달 닫기, 변경사항 없음

## 11. 비즈니스 규칙

### 11.1 가용 재고 계산

```
가용 재고 = 총 재고 - 예약 수량
```

**음수 가능:**

- 재고 조정으로 총 재고 < 예약 수량 상황 발생 가능
- 가용 재고 음수 표시 (예: -2개)
- 이 경우 "예약 초과" 또는 "재고 부족" 상태 표시 권장

### 11.2 Location 기반 재고 관리

**재고 수량 계산 원칙:**
```
재고 수량 = Location이 실제 창고인 자재의 합계
```

**재고 포함 Location:**
- 본사 창고
- 강남 창고
- AS 창고 (AS 재고로 별도 집계 가능)

**재고 제외 Location:**
- 현장 (출고됨)
- 불량창고 (검수 불량)
- 공급사 (입고 전)
- null (위치 미정)

**입고 시 재고 처리:**
1. 입고 등록 완료 시:
   - Location: 입고 창고로 설정
   - 재고 즉시 증가
   - 재고 변동 이력 생성

2. 검수 완료 시:
   - 정상품: Location 유지 (입고 창고)
   - 불량품: Location → 불량창고로 변경
   - 재고 수량: 정상품만 집계
   - 재고 변동 이력 업데이트 (사유/상태 추가)

**출고 시 재고 처리:**
- Location: 창고 → 현장으로 변경
- 재고 즉시 감소
- 재고 변동 이력 생성

**AS 입고 시 재고 처리:**
- Location: 현장 → AS 창고로 변경
- AS 재고 증가 (일반 재고와 별도 집계 가능)
- 재고 변동 이력 생성

### 11.3 재고 상태 판단 (3단계)

**1. 정상**

```
가용 재고 >= 안전재고
```

- 예약을 제외하고도 안전재고 이상 확보
- 초록색 뱃지

**2. 안전재고 미달**

```
0 <= 가용 재고 < 안전재고
```

- 예약은 충족 가능하지만 안전재고 미달
- 추가 입고 필요
- 주황색 뱃지

**3. 예약 초과 (심각)**

```
가용 재고 < 0
```

- 예약된 수량을 충족할 수 없는 심각한 상황
- 즉시 입고 또는 발주 취소 필요
- 빨간색 뱃지
- 가용 재고 빨간색 굵은 글씨

**예시:**

- 총 10개, 예약 5개, 가용 5개, 안전재고 3개 → **정상**
- 총 10개, 예약 8개, 가용 2개, 안전재고 3개 → **안전재고 미달**
- 총 10개, 예약 11개, 가용 -1개, 안전재고 3개 → **예약 초과**

### 11.4 재고 조정과 예약 수량

- 재고 조정 시 예약 수량은 변동 없음
- 재고 조정은 물리적 재고 수량만 변경
- 예약 수량은 발주 등록/취소 시에만 변동

### 11.5 재고 조정 이력

- 모든 조정 건은 재고 변동 이력에 자동 기록
- 조정 전/후 수량, 조정자, 조정일시 저장

### 11.6 시리얼 상태 변경 시 Location 및 재고 처리

**시리얼 상태와 Location 매핑:**

| 시리얼 상태 | Location | 재고 포함 여부 |
|------------|----------|---------------|
| 정상 | 본사창고 / 강남창고 | O |
| 외관 파손 | 본사창고 / 강남창고 | O |
| 기능 불량 | 불량창고 | X |
| 완전 파손 | 불량창고 | X |
| 분실 | null | X |
| 폐기 | null | X |
| 출고됨 | 현장명 | X |
| AS 중 | AS창고 | △ (AS 재고) |

**상태 변경 시 Location 처리:**

1. **정상 → 기능불량/완전파손**
   - Location: 창고 → 불량창고
   - 재고: -1개
   - 재고 변동 이력 생성: "상태변경: 기능불량"

2. **정상 → 분실**
   - Location: 창고 → null
   - 재고: -1개
   - 재고 변동 이력 생성: "분실: 현장분실"

3. **정상 → 폐기**
   - Location: 창고 → null
   - 재고: -1개
   - 재고 변동 이력 생성: "상태변경: 폐기"

4. **정상 → 외관파손**
   - Location: 창고 유지
   - 재고: 변동 없음
   - 재고 변동 이력 생성 안 함

5. **기능불량 → 정상 (수리 완료)**
   - Location: 불량창고 → 본사창고
   - 재고: +1개
   - 재고 변동 이력 생성: "상태변경: 수리완료"

**재고 변동 이력 생성 규칙:**
- Location 변경이 있는 경우: 재고 변동 이력 생성
- Location 변경이 없는 경우: 시리얼 이력에만 기록
